
# Online Account Security Review

I reviewed the security settings of three online accounts and checked whether Multi-Factor Authentication (MFA), strong passwords, and recovery email options were configured.

| Account  | MFA Enabled? | Strong Password? | Recovery Email Configured? |
| -------- | ------------ | ---------------- | -------------------------- |
| Google   | Yes          | Yes              | Yes                        |
| GitHub   | Yes          | Yes              | Yes                        |
| LinkedIn | No           | Yes              | Yes                        |

## Findings

* Google account has MFA enabled, a strong password, and a recovery email configured.
* GitHub account has MFA enabled and follows good security practices.
* LinkedIn account has a strong password and recovery email, but MFA is not enabled.

## Conclusion

Most of my accounts follow good security practices by using strong passwords and recovery email addresses. However, any account without MFA should be updated immediately because MFA provides an additional layer of protection. Using strong passwords, enabling MFA, and configuring recovery options helps keep online accounts secure from unauthorized access.
